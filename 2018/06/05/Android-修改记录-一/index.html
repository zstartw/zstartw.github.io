在低版本的手机上会出现GridLayout设置LayoutParams异常

解决:

aar 中的类找不到

现象:公司的app引用了一个第三方的库,新的需示是要更新这个库的版本,官方下载新的库覆盖上去,但是aar库没有生效
解决:找到第三方库对应的build.gradle,注释导入aar库的代码,再重新打开就OK

升级gradlew报错的问题

现象:之前项目都是用的gradlew 4.4,升级到4.6的时候报错,错误信息如下:

但是按照提示设置了org.gradle.configureondemand=false并没有起作用

解决:

  • 在gradle.properties删除org.gradle.configureondemand
  • Linux: File > Settings > Build, Execution, Deployment > Compiler，去掉configure on demand的勾选

Lint工具优化

每次提交有ImageView的配置文件的时候都会提示 “Image without contentDescription”警告,禁用的办法:

File > Settings > Editor > Inspections,去掉Image without contentDescription勾选

?attr 和?android:attr的区别

通过设置我发现:
?attr: 调用的是引入库里面的代码(.gradle/caches/transforms-1/files-1.1/appcompat-v7-27.1.1.aar)
?android:attr: 调用的是Android SDK 里面的代码(android-sdk/platforms/android-27/data/res/values/attrs.xml)

断点调试的时候报 no such instance field 提示

加了混淆minifyEnabled影响断点调试,设置minifyEnabled=false
